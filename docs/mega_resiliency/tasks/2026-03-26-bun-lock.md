# Wave 1 Task: Add bun.lock to Version Control (TD5, QW7)

**Date:** 2026-03-26
**Status:** Implemented
**Audit items:** TD5, QW7

## Finding

`bun.lock` was listed in `.gitignore`, meaning dependency resolution was non-reproducible
across machines. `bun install` could pick up different transitive dependency versions,
leading to builds that differ from CI or teammates' machines.

## Change

1. Removed `bun.lock` from `.gitignore`
2. Ran `/root/.bun/bin/bun install` to generate a fresh `bun.lock`
3. `bun.lock` is now tracked in git

## Lockfile contents

Two direct dependencies pinned:
- `diff@7.0.0`
- `playwright@1.58.2`

## Risk

Low. The lockfile ensures reproducible installs. Anyone running `bun install` now gets exactly
these versions. The only risk is if someone running an old bun version can't parse the lockfile
format, but bun lockfile format has been stable since v1.0.
